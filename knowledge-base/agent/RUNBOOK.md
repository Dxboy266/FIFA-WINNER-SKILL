Use this runbook when you are Codex, Claude Code, Cursor Agent, CI, or another runtime agent operating the repository.

1. `AGENT_README.md`
2. `knowledge-base/agent/AGENT_CARD.json`

4. `docs/runtime-agent-user-data-overlay-guide.md`

## Preflight

```bash
python -m pytest -q
```

If tests fail, report the failure before making prediction claims.


Use `prediction_scoring_model.py predict` or `octopus_paul_agent.py predict` for match/team/group/phase/all scoped work.

## User Data Overlay Flow

Use this flow when another runtime agent needs to use data generated by the user.

1. Generate or refresh user-local predictions:

```bash
python scripts/daily_prediction_runner.py run --edition 2026 --date 2026-06-11 --root .
```

2. Rebuild the dashboard merge view:

```bash
python scripts/prediction_visual_dashboard.py write --edition 2026 --root .
```

3. Read the dashboard JSON:

```text
knowledge-base/2026/data/reports/dashboard/prediction-dashboard.json
```

4. Interpret every card by `prediction_origin`:

- `user_local`: this is a user-generated prediction and overrides the bundled default.
- `octopus_default`: this is the bundled AI Octopus default prediction because the user has not generated that match locally.
- `none`: this is a public fact card only; show fixture or final-score facts, but do not claim a prediction exists.

5. When summarizing to the user, say which layer was used. Example:

```text
Source layer: user_local
Report: knowledge-base/2026/data/reports/daily-predictions/2026-06-11.json
Dashboard: knowledge-base/2026/data/reports/dashboard/prediction-dashboard.json
```

Do not edit public default predictions to represent user work. User-owned data belongs under `knowledge-base/<edition>/data/`.

## Summary Flow

Read the generated prediction report JSON and summarize:

- status
- report path
- match count
- main pick and score
- confidence
- evidence gaps
- top analysis layers
- safety disclaimer

Do not paste the full report unless the user asks for raw JSON.

## Poster Flow

Only run this when the user asks for visual material:

```bash
python scripts/poster_prompt_builder.py build --edition 2026 --date 2026-06-11 --style showdown --match-id 2026-GA-01 --root .
python scripts/poster_generator.py generate --manifest <poster-manifest.json> --backend image2 --root .
```

If the backend is missing, the blocked result is correct.

## Evaluation Flow

After final scores are recorded:

```bash
python scripts/prediction_evaluator.py write --edition 2026 --date 2026-06-11 --root .
python scripts/prediction_evaluation_dashboard.py write --edition 2026 --root .
```

Use evaluation output for calibration and reflection. Never rewrite pre-match picks after kickoff.
